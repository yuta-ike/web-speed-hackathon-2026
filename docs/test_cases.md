## 手動テスト項目

アプリケーションに機能落ちがないかどうかを確認するための手動テスト項目です。
競技終了後のレギュレーションチェックでは、これらの項目をもとに動作の確認を行います。

### ホーム

- タイトルが「タイムライン - CaX」となること
- タイムラインが表示されること
- 動画が自動で再生されること
- 音声の波形が表示されること
- 写真が枠を覆う形で拡縮していること
- 投稿をクリックすると、投稿詳細に遷移すること

### 投稿詳細

- タイトルが「{ユーザー名} さんのつぶやき - CaX」となること
- 投稿が表示されること
- Show Translation をクリックすると、投稿内容が英語に翻訳されること
- 翻訳後の投稿が表示されているとき、Show Original をクリックすると元の投稿文が表示されること

#### 投稿詳細 - 動画

- 動画が自動で再生されること
- 動画をクリックすると、一時停止・再生が切り替わること
- 再生される動画が著しく劣化していないこと（激しいブロックノイズが存在しないなど）

#### 投稿詳細 - 音声

- 音声の波形が表示されること
- 音声の再生ボタンをクリックすると、一時停止・再生が切り替わること
- 音声の再生位置が波形で表示されること

#### 投稿詳細 - 写真

- 写真が枠を覆う形で拡縮していること
- 画像が著しく劣化していないこと（激しいブロックノイズが存在しないなど）
- 「ALT を表示する」ボタンを押すと、画像の ALT が表示されること

### ユーザー詳細

- タイトルが「{ユーザー名} さんのタイムライン - CaX」となること
- ページ上部が、ユーザーサムネイル画像の色を抽出した色になっていること
- サービス利用開始の日時が正しく表示されること

### 利用規約

- タイトルが「利用規約 - CaX」となること
- "源ノ明朝（Noto Serif JP）" がインストールされている環境下で、フォントの表示が初期仕様と同じ見た目になっていること

### サインイン・新規登録

- 新規登録
  - ユーザー名が初期仕様通りにバリデーションされること
  - パスワードが初期仕様通りにバリデーションされること
  - サインインが成功するとモーダルが閉じ、サイドバーにサインアウトボタンが出現すること
  - 既に存在するユーザー名で登録しようとすると、エラーメッセージが表示されること
- サインイン
  - ユーザー名が初期仕様通りにバリデーションされること
  - パスワードが初期仕様通りにバリデーションされること
  - サインインが成功するとモーダルが閉じ、サイドバーにサインアウトボタンが出現すること
  - サインインに失敗するとエラーメッセージが表示されること
- サインアウト
  - サインアウトが成功するとサイドバーにサインインボタンが出現すること

### 投稿機能

- テキスト
  - テキストが投稿ができること
- 画像の投稿
  - TIFF 形式の画像を投稿できること
  - 画像の EXIF に埋め込まれた Image Description が ALT として表示されること
- 音声の投稿
  - WAV 形式の音声を投稿できること
  - Shift_JIS で付与された作成者・タイトルのメタデータが文字化けせずに表示されること
- 動画の投稿
  - MKV 形式の動画を投稿できること
  - 投稿した動画が先頭から5秒間のみに切り抜かれること
  - 投稿した動画が正方形に切り抜かれること

> [!TIP]
>
> テストに使用できる画像、音声、動画のサンプルファイルは [./assets](./assets) に置いてあります

### DM機能

- サインイン済みの場合、サイドバーに DM のリンクが表示されること
- 未サインインの場合、DM のリンクが表示されないこと
- DM一覧画面
  - タイトルが「ダイレクトメッセージ - CaX」となること
  - メッセージの送受信をしたことのある全てのユーザーが一覧表示されること
  - 表示順は最後にやり取りしたユーザーが上位になるようにソートされること
  - ユーザー名、プロフィール画像、最新のメッセージ本文、最終やり取りからの経過時間が表示されること
  - 未読のメッセージがある場合は「未読」のバッジが表示されること
  - 新規メッセージを受信した場合、画面がリアルタイムで更新されること
- 新規DM開始モーダル
  - ユーザー名の入力が初期仕様通りにバリデーションされること
  - 送信ボタンをクリックすると、DM詳細画面に遷移すること
  - 存在しないユーザー名の場合はエラーメッセージが表示されること
- DM詳細画面
  - タイトルが「{相手のユーザー名} さんとのダイレクトメッセージ - CaX」となること
  - メッセージの送受信履歴が表示されること
  - メッセージは古い順に表示されること
  - メッセージにはメッセージ本文、送信時間が表示されること
  - 初期表示で画面が一番下までスクロールされていること
  - 自分のメッセージで相手が既読している場合は「既読」のラベルが表示されること
  - 未読のメッセージがある状態でDM詳細画面を開いた場合、未読のメッセージが既読になること
  - 新規メッセージを受信した場合、画面がリアルタイムで更新されること
  - 新規メッセージを受信した場合、リアルタイムで相手のメッセージ履歴が既読になること
  - 新規メッセージを受信した場合、画面が一番下までスクロールされること
- 送信フォーム
  - メッセージの入力が初期仕様通りにバリデーションされること
  - Enterでメッセージを送信できること
  - Shift+Enterで改行できること
  - メッセージを入力中の場合、相手に入力中のインジケータが表示されること
  - メッセージの入力を辞めると入力中のインジケータが非表示になること
- 未読バッジ
  - 未読のメッセージがある場合はメニューに未読数のバッジが表示されること
  - 99件を超える場合は「99+」と表示されること
  - どのページからでも未読数がリアルタイムで更新されること

### Crok（AIチャット）

- タイトルが「Crok - CaX」となること
- サインイン済みの場合、サイドバーにCrokのリンクが表示されること
- 未サインインの場合、Crokのリンクが表示されないこと
- 初回表示時にウェルカム画面「AIアシスタントに質問してみましょう」が表示されること
- メッセージ入力欄にテキスト「TypeScriptの型」を入力すると、サジェスト候補が10件表示されること
  - サジェスト候補は入力内容に基づいてリアルタイムに絞り込まれること
  - サジェスト候補をクリックすると、入力欄にテキストが反映されること
  - サジェスト候補にマッチした名詞がハイライトされること
- メッセージを送信すると、AIのレスポンスが Server-Sent Events を用いてストリーミング表示されること
  - レスポンスが返却されている間は、送信ボタンが無効化されること
- AIのレスポンスにMarkdownが正しくレンダリングされること
  - [crok-response.md](../application/server/src/routes/api/crok-response.md) に記載された内容がレンダリングされること
  - コードブロックがシンタックスハイライトされること
  - 数式が初期仕様と同じ見た目でレンダリングされること
- Enterでメッセージを送信できること
- Shift+Enterで改行できること

### 検索

- タイトルが「検索 - CaX」となること
- 検索フォームが表示されること
  - 入力欄のプレースホルダーが「検索 (例: キーワード since:2025-01-01 until:2025-12-31)」であること
  - 「since:YYYY-MM-DD で開始日、until:YYYY-MM-DD で終了日を指定できます」のヘルプテキストが表示されること
- 検索クエリが初期仕様通りにバリデーションされること
- キーワードで検索すると、検索結果が表示されること
  - 検索条件と件数を含む見出しが表示されること
- キーワードと `since:YYYY-MM-DD` を組み合わせて検索できること
- キーワードと `since:YYYY-MM-DD` `until:YYYY-MM-DD` を組み合わせて検索できること
- 検索結果が見つからない場合、「検索結果が見つかりませんでした」と表示されること
- ネガティブなキーワードで検索した場合、「どしたん話聞こうか?」のメッセージが表示されること
  - ネガティブ判定には [単語感情極性対応表](http://www.lr.pi.titech.ac.jp/~takamura/pndic_ja.html) が使用されていること
- 検索結果のタイムラインが無限スクロールで追加読み込みされること

### レスポンシブ

- スマホ表示で表示が崩れないこと
- デスクトップ表示で表示が崩れないこと
